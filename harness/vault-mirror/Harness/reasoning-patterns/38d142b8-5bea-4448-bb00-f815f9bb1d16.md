User complaint framed as missing UI options (Composer 2.5 Fast toggle, Opus-style context window, subagent model picker). Research started from Obsidian task note and parallel semantic/grep searches for model/context/subagent wiring. Direction stayed research-only per role boundary; no implementation attempted. Investigation followed the request path top-down: TaskPanel model dropdown → task.model_id in TS/Rust/DB → pipeline_client resolve_model → scripts/pipeline-lib.mjs body.model → sidecar/handlers/pipeline.mjs hybrid branch → harness-orchestrator.ts SDK path → .cursor/agents/*.md frontmatter. Critical insight: agent frontmatter model (e.g. composer-2.5 on researcher-agent) is not consulted on the OpenClaw hybrid path—the task model wins for every specialist step. Secondary finding: @cursor/sdk options.d.ts exposes context/maxMode variants but SISpace does not surface them. Remaining uncertainty: whether @cursor/sdk Agent.create supports Composer context-window variants; whether swarm dispatches should share subagent_model_id; backward-compat behavior for existing tasks with only model_id.
