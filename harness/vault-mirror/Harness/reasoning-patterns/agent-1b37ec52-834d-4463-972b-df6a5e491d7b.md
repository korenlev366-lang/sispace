Root cause for stream cut-off was calling replaceLastLine on every SDK text delta (Ink re-render pressure), a post-stream 4000-character truncation, and wrap=truncate on chat lines. Root cause for image paste was pasting clipboard/terminal blobs directly into the input string without binary detection. Fix: accumulate stream text in streamTextRef, debounce UI updates via scheduleAgentStreamFlush (50ms), clear timer and flush full result.text on stream end; switch chat lines to wrap=wrap. For images: extractImageFromBlob/ingestPasteBlob with magic-byte and data-URL detection, session-scoped imageStoreBySession, placeholder substitution in send-turn via resolveImagePlaceholders after checking SDK image payload types. No failed attempts or user corrections; uncertainty remains around live TUI manual test of very long streams and drag-drop paste (only Ctrl+V and bracketed paste paths wired).
