### PROPOSAL-20260722-001

- Source task: post-task SDK chain
- Reason: The proposal generalizes the EPIPE guard pattern to all stream writes, preventing future similar crashes. It is clear, feasible, includes a rollback plan, and aligns with the task goal. The code changes and verification evidence demonstrate the approach works. Recommendation: accept.
- Target layer: code
- Date: 2026-07-22
- Rollback note: Revert `streamSafeWrite` usage and restore original write calls; delete the utility function.
- Applied change:

Generalize the EPIPE guard pattern to all stream writes in the CursorSI shell executor: create a reusable `streamSafeWrite` utility that prepends a no-op error listener before calling write, and audit all write calls in `src/shell-executor.ts`, `src/terminal/zsh-state.ts`, `src/terminal/bash-state.ts` to use this utility. This prevents future uncaught exceptions from other stream errors (ECONNRESET, EIO, etc.) in the same fashion.
